# 刷题助手

一个纯前端刷题网站，适合把 Excel 题库发给同学使用。项目不需要后端、不需要登录，题库和做题记录都保存在每个用户自己的浏览器 IndexedDB 中。

## 功能

- 支持导入 `.xlsx` 题库
- 内置默认题库，首次打开会自动加载
- 支持单题库答题和多题库混合考核
- 支持顺序答题、随机答题、限制抽题数量
- 支持自动判题、答案与解析展示
- 支持本次错题查看、错题重练、错题本、统计、本地数据清空
- 判断题可自动补全“正确 / 错误”选项
- 静态部署，服务器只需要托管 `dist/`

## 题库格式

推荐 Excel 表头：

| 列名 | 说明 |
| --- | --- |
| 题目 | 题干内容 |
| A、B、C、D、E、F | 选项内容，可按实际题目数量填写 |
| 答案 | 正确答案，例如 `A`、`AB`、`正确`、`错误` |
| 解析 | 题目解析，可为空 |
| 章节 | 章节名称，可为空 |
| 题型 | 例如 `单选题`、`多选题`、`判断题` |

判断题如果没有选项，系统会自动补为：

- `A. 正确`
- `B. 错误`

并把 `正确 / 对 / 是 / √ / true / 1` 标准化为 `A`，把 `错误 / 错 / 否 / × / false / 0` 标准化为 `B`。

## 本地开发

```bash
npm install
npm run dev
```

## 构建

```bash
npm run build
```

构建产物在 `dist/`，可以直接用 Nginx、Caddy、静态空间或对象存储托管。

## 部署说明

服务器只需要提供静态文件服务。以 Nginx 为例，把 `dist/` 内容上传到站点根目录，并使用 `try_files` 回退到 `index.html`：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

## 隐私说明

本项目第一版不包含账号、后端、云同步或排名功能。题库、答题记录、错题本都保存在本地浏览器中，不会上传到服务器。
